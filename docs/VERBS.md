This documents the **cmd/logger** reference binary `logger-demo` — a demonstration of the Mantle (github.com/inovacc/mantle) bootstrap runtime (cobra -> wrapper -> core app), not a product CLI. It exists to prove the end-to-end flow and to exercise every always-present persistent flag.

## Tree

```
logger-demo (root)
├── help        (cobra built-in) — Help about any command
└── completion  (cobra built-in) — Generate shell autocompletion script
```

The root command carries all 11 always-present persistent flags. The `help` and `completion` subcommands are cobra built-ins and inherit those persistent flags.

### logger-demo

**Synopsis:** Demonstrates cobra -> bootstrap wrapper -> core app.

**Usage:**

```
logger-demo [flags]
```

**Flags:**

| Flag | Default | Description |
|------|---------|-------------|
| `-c`, `--config` | (none) | Path to config file (yaml/yml/json). When omitted, the loader auto-creates `config.yaml` in the current working directory. |
| `--env` | `production` | Deployment environment name (e.g. development, staging, production). |
| `--log-level` | `info` | Log level: `debug`, `info`, `warn`, `error`. |
| `-v`, `--verbose` | `false` | Verbose output; raises the effective log level toward `debug`. |
| `-q`, `--quiet` | `false` | Quiet output; lowers the effective log level toward `error`. |
| `--log-format` | `json` | Log output format. Values: `json` \| `text`. |
| `--log-source` | `false` | Add source file/line (`AddSource`) to log records. |
| `--no-redact` | `false` | Disable PII redaction (redaction is on by default). |
| `--otel` | `false` | Enable OpenTelemetry (logs, traces, metrics) via the obsv stack. |
| `--otel-endpoint` | (none) | OTLP collector endpoint (e.g. `localhost:4317`). |
| `--otel-protocol` | `grpc` | OTLP transport protocol: `grpc` \| `http`. |
| `--version` | `false` | Print version and exit. |

Flags overlay file and environment configuration: only **changed** flags take effect, and they hold the highest precedence (defaults -> file -> env -> changed flags).

**Examples:**

```
logger-demo --otel --otel-endpoint localhost:4317
logger-demo -v --log-format text
logger-demo -c ./config.yaml
```

**Notes:**

- `--help` short-circuits before any config is loaded — running `logger-demo --help` never triggers config file creation or env resolution.
- Without `--config`, the inovacc/config loader auto-creates `config.yaml` in the current working directory on first run.
